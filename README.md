# Проект SCADA системы автополива (верхний уровень)

[![Language: C](https://img.shields.io/badge/Language-C-blue.svg)](https://en.wikipedia.org/wiki/C_(programming_language))

[![Display: 800×480 RGB](https://img.shields.io/badge/Display-800%C3%97480%20RGB-orange.svg)](https://www.waveshare.com/esp32-s3-touch-lcd-7.htm)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Верхний уровень SCADA системы для управления автополивом на базе платы **Waveshare ESP32-S3-Touch-LCD-7 (Rev 1.2)** с сенсорным дисплеем 800×480 (RGB интерфейс, GT911). Реализует:

- **Точку доступа Wi‑Fi (SoftAP)** с TCP-сервером для связи с нижними устройствами (ESP32-C3).
- **Клиентский режим Wi‑Fi (STA)** для получения точного времени через NTP.
- **Графический интерфейс** на LVGL v9.1 с отображением данных, графиков и управлением.
- **Управление подсветкой** через I2C-расширитель CH422G с возможностью автоотключения при бездействии.
- **Приём данных** от нижнего уровня (JSON, файлы логов) и построение графиков температуры, влажности, состояний насоса и клапана.
- **Удалённое управление** насосом и клапаном через кнопки в UI.

## ⚙️ Настройка (Kconfig)

Перед сборкой выполните `idf.py menuconfig` и установите параметры:

### Wi-Fi SoftAP Configuration
- **SSID** – по умолчанию `ESP32_S3_Display`
- **Password** – по умолчанию `12345678`
- **Channel** – 1-11 (по умолчанию 6)

### Wi-Fi Client (STA) Settings (для синхронизации времени)
- **Enable Wi-Fi Station (client) mode** – включите, чтобы ESP32 подключался к вашей домашней сети
- **Wi-Fi SSID** – название вашей сети 
- **Wi-Fi Password** – пароль от сети
- **NTP Server** – по умолчанию `pool.ntp.org` (можно заменить на `ru.pool.ntp.org` или `time.google.com`)
- **Timezone offset (hours from UTC)** – часовой пояс (например, для Москвы `3`)

### Backlight Settings
- **Backlight timeout (seconds)** – время бездействия, после которого подсветка выключается (по умолчанию 30 сек)

Остальные параметры (LVGL, PSRAM, SPIFFS) оставьте по умолчанию или настройте под свои нужды.

## 🔧 Сборка и прошивка

1. Установите ESP-IDF (ветка `v6.1-dev` или `v6.2`).  
   Подробнее: [ESP-IDF Getting Started](https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/get-started/index.html)

2. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/Raven078/arrigation_system.git
   cd arrigation_system
Настройте проект через menuconfig (выберите целевой чип ESP32-S3):

bash
idf.py set-target esp32s3
idf.py menuconfig
Соберите и прошейте:

bash
idf.py clean build flash monitor
После успешной прошивки устройство создаст точку доступа ESP32_S3_Display и, если STA включена, подключится к вашей внешней сети для получения времени.

# Использование

## Главный экран
Отображает фоновое изображение (800×480) и текущее время/дату (обновляется каждую минуту).

## Три кнопки: «Точка доступа», «Помидорник», «Огуречник» (пока реализован только помидорник).

### Окно «Точка доступа»
Показывает SSID, IP (192.168.4.1), порт (8888) и пароль (по умолчанию скрыт звёздочками, показывается по нажатию).

Список подключённых клиентов с MAC-адресами и предполагаемыми IP.

### Окно «Помидорник»
Отображает текущие показания: температура, влажность, время последнего обновления.

Индикаторы датчиков уровня (верхний и нижний).

Кнопки для ручного управления насосом (ON/OFF) и клапаном (ON/OFF).

Регулировка скорости насоса (кнопки + и -, шаг 5%).

Два графика:

Основной – температура (зелёный) и влажность (синий) за последние 24 часа (144 точки, ось времени – каждые 10 минут).

Дополнительный – состояние насоса (красный) и клапана (оранжевый), значения 80% (вкл/открыт) и 0% (выкл/закрыт).

Окно автоматически закрывается через 10 минут бездействия или при потере связи с клиентом (проверка каждые 78 секунд).

Взаимодействие с нижним уровнем
Нижние устройства (ESP32-C3, репозиторий arregation_system_tomato) подключаются к точке доступа верхнего уровня.

После подключения они отправляют команду CMD_PORT:8889, получают текущее время от TCP-сервера и регистрируются.

Далее обмен данными происходит по двум каналам:

JSON – передача текущих значений датчиков (температура, влажность, уровни, состояния насоса/клапана, скорость).

Файлы – каждые 10 минут (в минуты, кратные 10) нижний уровень отправляет файл с историей за день (greenhouse_ДД_ММ_ГГГГ.txt).

При получении файла верхний уровень обновляет графики.

📈 Логирование и графики
Данные сохраняются в SPIFFS в файлы /spiffs/pomodoro_ДД_ММ_ГГГГ.txt.

При старте системы все старые лог-файлы удаляются (гарантирует актуальность данных).

Графики строятся по данным из последнего полученного файла.

Обновление графиков происходит автоматически при получении нового файла от клиента.

🛠️ Отладка
Для просмотра логов используйте idf.py monitor (порт /dev/ttyACM0, скорость 115200).

Ключевые теги:

main – запуск и инициализация

WIFI_AP – события Wi‑Fi (SoftAP и STA)

TCP_SERVER – работа TCP-сервера, приём команд

UI_POMODORO – отладка интерфейса помидорника

RTC_TIME – синхронизация времени (NTP)

BACKLIGHT_CH422G – управление подсветкой

📦 Зависимости (управляемые компоненты)
Проект использует idf_component.yml и включает:

LVGL v9.1 – графический интерфейс

esp_lvgl_port – интеграция с ESP-IDF

esp_lcd_st7701 – драйвер дисплея

esp_lcd_touch_gt911 – драйвер сенсора

cJSON – парсинг/генерация JSON

🖼️ Создание и замена фонового изображения
Фоновое изображение хранится в виде C-массива в main/GUI/wallpaper/wallpaper800400.c. Для его перегенерации используйте официальный LVGL Image Converter или локальный LVGLImage.py (из исходников LVGL).
Настройки:

Формат цвета: True color with Alpha (RGB565)

Размер: 800×480

Полученный .c-файл замените существующий в проекте.

✍️ Шрифты
В проекте используется пользовательский шрифт my_arial24.c (Arial 24 пикселя с поддержкой кириллицы). Для создания шрифтов используйте LVGL Font Converter или локально:

bash
npx lv_font_conv --bpp 4 --size 24 --format lvgl \
  --font /путь/к/шрифту.ttf \
  -r 0x20-0x7F,0x410-0x44F,0x401,0x451 \
  --lv-font-name my_arial24 \
  --no-compress \
  -o my_arial24.c
Важно: параметр --no-compress обязателен для корректного отображения в LVGL.
