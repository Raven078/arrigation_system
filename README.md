# Проект ESP32-S3 Touch LCD 7 с точкой доступа и мониторингом теплиц
[![ESP-IDF](https://img.shields.io/badge/ESP--IDF-v6.1-blue)](https://github.com/espressif/esp-idf)
[![Platform](https://img.shields.io/badge/platform-ESP32--S3-green)](https://www.espressif.com/en/products/socs/esp32-s3)
[![Language](https://img.shields.io/badge/language-C-00599C)](https://en.wikipedia.org/wiki/C_(programming_language))

Проект для платы Waveshare ESP32-S3 Touch LCD 7 с сенсорным дисплеем 800×480. Реализует:
- Точку доступа Wi‑Fi (SoftAP) с TCP‑сервером.
- Главный экран с тремя кнопками: **Точка доступа**, **Помидорник**, **Огуречник**.
- Окно информации о точке доступа: SSID, IP, порт, пароль (скрытый, показывается по нажатию), список подключённых устройств.
- Окна мониторинга для двух теплиц ("Помидорник" и "Огуречник") с графиками температуры и влажности, индикаторами датчиков уровня.

## Аппаратное обеспечение
- Плата: Waveshare ESP32-S3 Touch LCD 7 (800×480 RGB, сенсор GT911).
- ESP-IDF версия 6.1 (совместима с веткой разработки).
- Дисплей: RGB параллельный интерфейс, пины согласно репозиторию.
- PSRAM: 8 МБ (Octal).

## Возможности
- **Точка доступа**:
  - SSID: `ESP32_S3_Display`, пароль: `12345678`.
  - TCP-сервер на порту 8888.
  - Команды: `GET_TIME`, `GET_TIMESTAMP` (возвращает текущее время).
  - Окно с информацией и списком клиентов (MAC + предположительный IP).
- **Помидорник / Огуречник**:
  - Полноэкранные окна с бледно-голубым фоном.
  - Верхняя часть (220px) – прямоугольник с двумя индикаторами (кружки) для датчиков уровня.
  - Нижняя часть – график (ширина 710, высота 200) с двумя осями:
    - Левая ось: влажность 0–100% (синий график).
    - Правая ось: температура 0–50 °C (красный график, масштаб данных *10).
  - Ось времени: 24 часа, подписи часов, риски через 10 минут.
  - Кнопка закрытия (буква **X** шрифтом `my_arial24`).
  - Обновление данных через вызов `ui_pomodoro_update_sensors()` и `ui_cucumber_update_sensors()` из TCP-обработчика.
- **В дальнейшем развитии проекта будет ввыполнено получение данных о температуры и влажности, а также о состоянии датчиков уровня, 
времени ввключения насоса и открытия и закрытия клапана  с устройств Огуречник и Помидорник в формате JSON c порта TCP, в который информация передается устройствами.
Устройства Огуречник и Помидорник выполнены на плате разработки ESP32c3 super mini. При развитии проекта будет добавлена ссылка на репозиторий.

## Сборка и прошивка

1. Установите ESP-IDF (ветка v6.1 или новее).
2. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/Raven078/waveshare-ESP32s3_touch_LCD7.git
   cd waveshare-ESP32s3_touch_LCD7
Настройте проект (при необходимости):

bash
idf.py menuconfig
Включите PSRAM: Component config → ESP PSRAM → Support for external, SPI-connected RAM.

Установите тип Octal и скорость 80MHz.

Соберите и прошейте:

bash
idf.py build
idf.py flash monitor
Использование
После запуска на дисплее отображается фоновое изображение, три кнопки и время/дата (правый нижний угол).

Кнопка "Точка доступа" открывает окно с информацией о сети. Список устройств обновляется по кнопке Обновить. Пароль скрыт звёздочками, показывается по нажатию.

Кнопки "Помидорник" и "Огуречник" открывают окна мониторинга. Графики и индикаторы обновляются при получении данных по TCP.

Для отправки данных в ESP32 используйте TCP‑клиент (например, nc 192.168.4.1 8888). Формат JSON (как в логах).

Пример данных для обновления
Вызов из TCP-обработчика:

c
if (strcmp(cmd, "sensors_pomodoro") == 0) {
    float temp = 24.5;
    int humi = 65;
    bool s1 = true, s2 = false;
    ui_pomodoro_update_sensors(temp, humi, s1, s2);
}
Аналогично для огуречника: ui_cucumber_update_sensors(...).

Примечания
Фоновое изображение хранится во flash (массив 768 КБ). Для его замены перегенерируйте wallpaper800400.c из PNG/RGB565.

Шрифт my_arial24 – Arial 24 пикселя, используется для кнопок и надписей.

Графики отображают 144 точки (24 часа × 6 точек/час). При поступлении новых данных массив сдвигается.

Точки (маркеры) на графиках скрыты через стиль LV_PART_INDICATOR.

🖼️ Создание и добавление фонового изображения
Фоновое изображение в проекте хранится в виде C-массива в файле main/GUI/wallpaper/wallpaper800400.c. Для его создания вы можете использовать как официальный онлайн-инструмент, так и локальный конвертер.

# Онлайн-конвертер (LVGL Online Image Converter)
Это самый простой способ быстро получить необходимый C-файл.
- Выберите изображение: Перейдите на официальный сайт и нажмите Select image file(s). Вы можете загрузить файлы в формате PNG, JPG, BMP и другие.
- Настройте параметры:
- Output file name: Укажите имя для выходного C-файла, например, wallpaper800400.
- Color format: Выберите True color with Alpha (RGB565) для RGB565. Это предпочтительный формат для LVGL.
- Конвертируйте: Нажмите кнопку Convert и сохраните полученный .c файл.
- Добавьте в проект: Скопируйте сгенерированный .c файл в папку вашего проекта (.../main/GUI/wallpaper/). Его структура уже будет содержать нужные массивы и дескриптор, который можно сразу        использовать в коде.

## Офлайн-конвертер (LVGLImage.py и lv_img_conv)
Если интернет недоступен или требуется автоматизация, используйте официальные локальные утилиты.

Простой вариант (LVGLImage.py):

Утилита находится в папке исходных кодов LVGL: ./scripts/LVGLImage.py.

- Запустите её без аргументов для получения полного списка опций: ./LVGLImage.py. Команда для конвертации может выглядеть так: LVGLImage.py -i input.png -o output.c --format RGB565.
- Гибкий вариант (lv_img_conv):
- Сначала установите Node.js, так как утилита написана на JavaScript.
- Клонируйте репозиторий: git clone https://github.com/lvgl/lv_img_conv.git.
- Запустите команду для конвертации, например: ./lv_img_conv.js input.png output.c -f -c CF_TRUE_COLOR_ALPHA.
- Сгенерированный .c файл так же, как и в онлайн-версии, переносится в папку с проектом.

Примечание: Проект ожидает конкретную структуру массива в wallpaper800400.c. Убедитесь, что имя вашего массива соответствует дескриптору, на который ссылается UI (например, wallpaper800400).

✍️ Создание и использование пользовательского шрифта
В проекте, наряду со стандартными шрифтами LVGL, используется пользовательский шрифт my_arial24. Вы также можете создать и добавить свои шрифты для других языков и размеров.

# Онлайн-конвертер (LVGL Online Font Converter)
## Зайдите на официальную страницу конвертера.

Заполните поля настройки:

- Name: Задайте имя для генерируемого C-файла и переменной шрифта (например, my_arial_24).
- Size (px): Укажите желаемую высоту шрифта в пикселях.
- bpp (Bits per pixel): Выберите 4 для хорошего сглаживания (анти-алиасинга).
- Font: Загрузите ваш .ttf или .woff файл с помощью кнопки Choose a TTF or WOFF font.
- Range / Symbols: Укажите нужные символы. В поле Range можно задать диапазоны Unicode, например 0x20-0x7F для латиницы.
- Нажмите Convert, чтобы скачать сгенерированный C-файл.

## Офлайн-конвертер (lv_font_conv)
Самый мощный способ, особенно для автоматизации. Требует установки Node.js.

### Установка инструментов для шрифтов Для генерации собственных шрифтов (с поддержкой кириллицы) потребуется Node.js и пакет lv_font_conv.

Установка Node.js и npm bash sudo apt update sudo apt install nodejs npm

### Локальная установка lv_font_conv в папке проекта bash cd /home/evgeny/esp/project/lvgl_color_squares npm install lv_font_conv После этого команда будет доступна через npx:

Для поддержки кириллицы используется шрифт my_arial, сгенерированный с помощью lv_font_conv. Очень важный параметр --no-compress — без него шрифт может отображаться некорректно (текст может пропадать или вызывать ошибки LVGL).

bash

npx lv_font_conv --bpp 4 --size 24 --format lvgl \
--font "/home/evgeny/esp/project/отдельные элементы кода для дисплея/arial-regular.ttf" \
-r 0x20-0x7F,0x410-0x44F,0x401,0x451 \
--lv-font-name my_arial24 \
--no-compress \
-o my_arial24.c

Важно! В данном проекте для отсутствия заморочек с использоанием строенных шрифтов LVGL данным скриптом был сгенерирован шрифт lv_montserrat14 и заменен массив глифов во встроенном шрифте LVGL по адресу ~/managed_components/lvgl__lvgl/src/font/ такое не приветствуется, но мне так было проще!


Важно: При использовании сгенерированного файла в коде проекта (ui.c, ui1.c) необходимо объявить его как внешнюю переменную:

c
// Например, в ui.c
LV_FONT_DECLARE(my_arial_24);
// Затем использовать в стиле
lv_obj_set_style_text_font(object, &my_arial_24, 0);
